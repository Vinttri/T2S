# Бенч-приложение — гид для людей

Простыми словами: это сайт, где можно **сравнивать разные Text-to-SQL модели**.
Ты говоришь приложению «вот моя модель (по такому-то адресу)» и «вот набор
вопросов с правильными SQL-ответами» — оно задаёт модели каждый вопрос, проверяет
её SQL на настоящей базе и ставит оценку. Потом всё это видно на лидерборде.

Открыть: **http://benchmark.144.91.85.207.nip.io:8080/**

---

## Главные понятия (на пальцах)

- **Коннектор** = одна модель. Это «как достучаться до API модели»: адрес, что
  отправить, откуда взять SQL из ответа. Один коннектор — на одну базу.
- **Датасет (бенчмарк)** = набор вопросов + правильные (эталонные) SQL + база, на
  которой всё проверяется.
- **Прогон** = «прогнали одну модель по одному набору вопросов». На каждый вопрос
  модель пишет SQL, мы его запускаем и сравниваем с эталоном.
- **Оценка L0–L4** — насколько ответ хорош:
  - **L4** — идеально, результат совпал с эталоном;
  - **L3** — SQL выполнился, но ответ не тот (логическая ошибка);
  - **L2** — упал сам эталонный запрос (проблема не модели);
  - **L1** — SQL не выполнился (синтаксис, нет таблицы и т.п.);
  - **L0** — модель вообще не дала SQL.
  «Точность» (accuracy) = доля L4.

## Вкладки

**🏆 Лидерборд.** Сравнение моделей по выбранному бенчмарку. Видно точность,
графики, и таблицу «вопрос × модель» с оценками. Можно:
- фильтровать модели (пилюли сверху),
- у каждой модели выбрать, **какой прогон** показывать (по умолчанию — крайний),
- нажать **«подробнее»** у вопроса → увидеть эталонный ответ и ответы всех моделей,
- скачать результаты кнопкой **⬇ JSON**.

**🔌 Коннекторы.** Здесь создаёшь/редактируешь модели. Заполняешь адрес, тело
запроса (с подстановкой `{{question}}`), откуда брать SQL. Кнопка **«Тест»**
дёрнет модель реальным вопросом из её бенчмарка и покажет, какой SQL она вернула;
**«Превью»** — просто покажет, какой запрос уйдёт, ничего не отправляя.

**▶️ Запуск.** Выбираешь датасет и коннектор → «Запустить». Если модель и база
несовместимы (разные движки или разные базы) — кнопка заблокируется и подскажет.

**📈 Тесты.** Все прогоны живым списком (обновляется сам). Сгруппированы на
«выполняется» и «завершено». У каждого прогона — прогресс, кнопки **пауза / стоп**
и **«перепрогнать невыполненные»** (если часть вопросов упала из-за сбоев). Клик по
вопросу раскрывает детали (эталон vs ответ модели), а **↻** перепрогоняет один
вопрос (рядом крутится «перезапуск…»).

**📊 Результаты.** Выбрать датасет и ревизию (по умолчанию крайнюю), посмотреть
подробности по каждому вопросу и скачать JSON.

**📝 Обзоры решений.** Текстовые описания моделей — как устроены, плюсы/минусы.
Можно редактировать прямо на сайте.

Сверху справа — индикатор связи (WebSocket) и время последнего обновления. Тема
(светлая/тёмная) и текущая вкладка запоминаются.

## Как добавить свою модель и прогнать (по шагам)

1. **🔌 Коннекторы → «Новый коннектор».** Укажи название, адрес (URL), тело
   запроса с `{{question}}`, и откуда брать SQL (обычно — поле `sql` в JSON-ответе).
   Укажи **целевую БД** (db_id) — например `dm_mis`.
2. Нажми **«Тест»** — должен вернуться SQL. Если ок — **«Сохранить»**.
3. **▶️ Запуск** — выбери датасет этой БД и свой коннектор → **«Запустить прогон»**.
4. **📈 Тесты** — смотри живой прогресс. Упало из-за сбоя сети/сервиса — нажми
   **«перепрогнать невыполненные»**.
5. **🏆 Лидерборд** — результат появится в сравнении; можно скачать JSON.

## Частые вопросы

- **Почему модель показывает 0%?** Скорее всего её бэкенд не знает эту базу (отвечает
  «Unknown database») или сервис недоступен. Это честный результат — видно сразу.
- **Почему так медленно?** Сами модели думают долго (от десятков секунд до минут).
  Прогон идёт в фоне — можно закрыть вкладку и вернуться.
- **Можно ли поправить модель/описание?** Да: коннекторы редактируются на вкладке
  «Коннекторы», обзоры — на «Обзоры решений».
