Датасеты

Датасет — это JSONL-файл `BENCHMARK_*.jsonl`: одна строка JSON на один вопрос + gold SQL. `db_id` связывает датасет с коннектором, `Тип БД` задаёт SQL-диалект для проверки совместимости. DSN scoring-базы backend берёт только из env по `db_id/db_type`.
Пример формата benchmark JSONL
{"benchmark_id":"S1","case_id":"dm_mis_1","difficulty":"Simple","question":"Напиши запрос, который выведет исходящий остаток по счету.","normal_phrasing":"","conditions":"","gold_sql":"SELECT balance_out\nFROM dm_mis.v_f_account_rest\nWHERE report_date = DATE '2026-05-24'\nLIMIT 100;"}
+ добавить датасет
Файл будет сохранён автоматически в runtime-хранилище приложения (`/data/datasets` в Docker).
DSN не вводится в UI: задайте env вроде BENCH_<DB_ID>_<DB_TYPE>_DSN, BENCH_<DB_ID>_DSN или BENCH_SCORING_DSN.
